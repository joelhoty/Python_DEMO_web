What happens when you submit:
  1. The backend reads your username from the teaching_user cookie.
  2. It fetches your stored hash from the database and verifies your current password.
  3. If correct, it generates a new hash for the new password and updates the database.
  4. If incorrect, it returns an error — the old hash is kept unchanged.
