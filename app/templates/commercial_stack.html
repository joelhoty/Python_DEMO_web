Commercial Stack Overview

Concept: Real production websites are built from many specialised tools. Here is how a typical frontend, backend, database, and server stack fits together.

Frontend (what runs in the browser)

TechnologyRole
ReactJavaScript UI library from Meta — builds interactive component trees
VueProgressive JavaScript framework — easier learning curve than React
AngularFull TypeScript framework from Google — opinionated, batteries-included
Next.jsReact with server-side rendering and routing built in

Backend (what runs on the server)

TechnologyRole
FastAPI (Python)Modern async Python framework — used by this demo site
Django (Python)Full-stack Python framework with ORM, admin, and auth built in
Node.js / ExpressJavaScript on the server — same language as the frontend
Ruby on RailsConvention-over-configuration Ruby framework
Spring Boot (Java)Enterprise Java framework with extensive ecosystem

Database (where data is stored)

TechnologyRole
PostgreSQLOpen-source relational database — common choice for production
MySQL / MariaDBAnother popular relational database
SQLiteFile-based relational database — used by this demo for simplicity
MongoDBDocument-oriented (NoSQL) — stores JSON-like documents
RedisIn-memory key-value store — often used for caching or sessions

Servers & Hosting

TechnologyRole
Nginx / ApacheWeb server — handles static files and proxies to the backend
AWS / GCP / AzureCloud platforms — provide compute, storage, and managed services
DockerContainer runtime — packages the app and its dependencies together
KubernetesContainer orchestration — scales and manages containers in production
How they fit together: A user visits a React frontend, which calls a FastAPI backend, which reads from PostgreSQL. Everything runs inside Docker containers on AWS. This demo uses only FastAPI + SQLite to keep the setup simple and focus on core concepts.
